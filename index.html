    Thoughts on Progressive Enhancement

    My thoughts on progressive enhancement are that it is a web development strategy that focuses on delivering a functional and accessible experience for everyone, no matter the browser capabilities. Sort of like what we spoke about towards the beginning of the class with building solid foundations and functioning site using semantic HTML, adding in some CSS to crate the style (look) and finishing it off with JS for interaction. Similar to how the broke down the M&M peanut with the peanut representing the HTLM, the chocolate represents the CSS (the users don’t see what we do but they can see it with their experience), and then the hard coated candy which represents the JSI liked that example because I am one of those that eats the Peanut M&M’s in layers, I eat the hard part first then I eat the chocolate and then savor the peanut ad each layer adds some form of enhancement similar to coding.
